2019-11-29

2019-11-30

SETA の仕様変更: ポインタに対しては長さのチェックを行わない。

ライブラリからロード機能を追加。 下のほうにあるボタンにライブラリ名を入れて PadIt することでライブラリにある PAD データを一瞬でロードできる。 現時点で利用可能なライブラリ名: OddEvenSort, SelectionSort.

2019-12-01

新コマンド PRINTA 追加。 書式は PRINTA v [n]。 配列 v の要素をすべて、もしくは n 個表示。 v がポインタの場合は n は必須。

2019-12-02

メモリモリモリ事件の犯人が大体実行済みコマンド一覧であったことが明らかになったため、コマンド一覧は懲戒免職を受けた。 これによって全体的な実行速度が格段に向上したほか、メモリ利用量も改善した。 これで無限ループによるメモリ爆発はおこらなくなったはずである。

ライブラリに追加: InsertionSort。

新コマンド SWAPA 追加。 書式: SWAPA v n m。 配列 v のインデクス n, m 番目の要素を入れ替える。 異なる配列間の要素を入れ替えるには GETA を利用して要素のインデクスを取得したのちに MEMORY に対して本コマンドを利用するなどできる。

2019-12-03

新コマンド PUSH, POP 追加。 書式: PUSH x; POP a。 PUSH コマンドを用いてスタックに値を積み込みます。 POP コマンドを利用してスタックから値を取り出します。

2019-12-04

一部の変数名を利用するとシステム全体が不安定になる問題を修正。

条件式を要求するブロックを選択時に条件コマンド以外のコマンドを入力されると通常ブロックに自動的に切り替わるようになった。 これは(現時点では)内部的に切り替わるだけであって、ラジオボックスの値は切り替わらない。

条件式にのみ $ から始まる $ i == 0 のような構文が理解できるようになった。 これ以降でも $ から始まるコマンドは “自然な” 式を理解するコマンドとして利用できるようになるかもしれない。

ライブラリ OddEvenSort, SelectionSort を新構文を利用した形式に書き直しました。 InsertionSort にバグがあるようです。

2019-12-05

$ から始まるコマンドブロックをサポート。 単純代入(=, <-)、演算代入(+=, *=, &=, etc)とスワップ(<=>)が利用可能になりました。 単項増減分演算子(++, --)が利用可能になりました。 それぞれの演算子と演算項はスペースで分けてください(ex. $ i = 0, $ a <<= 3, $ i ++, etc)。

2019-12-06

Favicon を追加。 ごみのような出来栄え。

さりげなしにコメントブロックのデザイン変更。

新コマンド BREAK を追加。 ループ内で利用する場合はループを脱出する break と同様の機能。 ループ外で利用する場合(ルート直下やルート直下に配置された if など)はプログラムを脱出する。

新コマンド CONTINUE を追加。 ループ内で利用する場合は continue と同様の機能。 ループ外に利用する場合はプログラムを脱出する。

新コマンド INPUT を追加。 書式は INPUT a [b [...]]。 入力がキャンセルされるとシステム変数 RETURN にシステム定数 EOF が代入されます。

システム変数 RETURN を追加。 これには直前に実行したコマンド(や将来的には追加されるかもしれない関数)の返り値が格納されます。

システム定数 EOF を追加。 入力機能のエラーや中断時に返却されます。

テューリング完全性の確認作業が完了。 新ライブラリ: Rule110。
